# Stack patterns — Next.js API + React + PostgreSQL

## Directory structure (App Router)

```
src/
  app/
    api/             # Route handlers (server-side)
      users/
        route.ts     # GET /api/users, POST /api/users
      users/[id]/
        route.ts     # GET /api/users/:id, PUT, DELETE
    (pages)/         # Page components
      dashboard/
        page.tsx
      layout.tsx     # Root layout
  components/        # Reusable React components
  lib/
    db.ts            # Prisma client singleton
    auth.ts          # Auth helpers
  types/             # TypeScript type definitions
prisma/
  schema.prisma      # Data model — source of truth for DB
  migrations/        # Auto-generated by prisma migrate
.env.local           # DB connection, secrets — never commit
```

## Prisma — single client instance

```typescript
// src/lib/db.ts
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };
export const db = globalForPrisma.prisma ?? new PrismaClient();
if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = db;
```

Always import `db` from `@/lib/db`. Never instantiate `PrismaClient` directly in route handlers.

## API route handlers (App Router)

```typescript
// src/app/api/users/route.ts
import { NextResponse } from "next/server";
import { db } from "@/lib/db";

export async function GET() {
  const users = await db.user.findMany();
  return NextResponse.json(users);
}

export async function POST(req: Request) {
  const body = await req.json();
  const user = await db.user.create({ data: body });
  return NextResponse.json(user, { status: 201 });
}
```

## Prisma schema pattern

```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id         String      @id @default(cuid())
  email      String      @unique
  milestones Milestone[]
  createdAt  DateTime    @default(now())
}

model Milestone {
  id        String   @id @default(cuid())
  name      String
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  createdAt DateTime @default(now())
}
```

After any schema change: `npx prisma migrate dev --name <description>`

## React components — client vs server

Default to Server Components (no `"use client"`). Add `"use client"` only when you need:
- `useState` / `useEffect`
- Browser APIs
- Event handlers

```tsx
// components/MilestoneList.tsx — Server Component, no "use client"
import { db } from "@/lib/db";

export default async function MilestoneList() {
  const milestones = await db.milestone.findMany();
  return <ul>{milestones.map(m => <li key={m.id}>{m.name}</li>)}</ul>;
}
```

## Environment variables

- `DATABASE_URL` — Prisma DB connection string
- `NEXTAUTH_SECRET` — if using NextAuth
- `NEXTAUTH_URL` — base URL
- All secrets go in `.env.local` (gitignored), not `.env`

## Naming conventions

- Route files: always `route.ts` (not named after the resource)
- Page files: always `page.tsx`
- Components: PascalCase files — `MilestoneCard.tsx`
- Prisma models: singular PascalCase — `User`, `Milestone`
- DB fields: camelCase — `createdAt`, `userId`

## Common mistakes to avoid

- Do NOT create a new `PrismaClient()` in every route — use the singleton
- Do NOT use the Pages Router (`pages/`) — use App Router (`app/`)
- Do NOT use `"use client"` by default — start server-side, opt in as needed
- Do NOT commit `.env.local`
- Do NOT skip `prisma generate` after schema changes
